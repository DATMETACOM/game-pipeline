## 🎮 GAME CORE BREAKDOWN

### Core Mechanic
```
MATCH-3 + PATHFINDING = STRATEGIC PUZZLE

Tap tiles → Create match → Path generates → Hero moves → Reach exit
```

### What Makes It Unique
| Element | Standard Match-3 | Maze Match Hero | Difference |
|---------|-----------------|-----------------|------------|
| **Goal** | Clear all tiles | Reach exit | Spatial reasoning |
| **Progress** | Score/Stars | Path building | Visible progress |
| **Strategy** | Match anywhere | Plan path direction | Spatial planning |
| **Payoff** | Screen clear | Hero reaches goal | Character journey |

### Why This Works
- ✅ **Proven Core:** Match-3 = low CPI, high IAP potential
- ✅ **Fresh Twist:** Pathfinding = strategic depth, differentiation
- ✅ **Character:** Hero = emotional hook, memorable
- ✅ **Fair Challenge:** APS 1.4-1.8 = sweet spot for retention + monetization

## 📈 APS ANALYSIS (50 LEVELS)

### APS Curve Design

| Level Block | APS Target | Range | Levels | Success Rate | Purpose |
|-------------|------------|-------|--------|--------------|---------|
| **1-10** | 1.15 | 1.1-1.2 | 10 | 83-91% | Onboarding, teach mechanics |
| **11-30** | 1.40 | 1.3-1.5 | 20 | 67-77% | Core gameplay, strategic choice |
| **31-50** | 1.65 | 1.5-1.8 | 20 | 56-63% | Monetization, tight challenge |
| **OVERALL** | **1.45** | **1.4-1.8** | **50** | **69%** | **Target met ✅** |

### APS Calculation
```
APS = (APS₁₋₁₀ × 10 + APS₁₁₋₃₀ × 20 + APS₃₁₋₅₀ × 20) / 50
APS = (1.15 × 10 + 1.40 × 20 + 1.65 × 20) / 50
APS = (11.5 + 28.0 + 33.0) / 50
APS = 72.5 / 50
APS = 1.45 ✅

Requirement: APS > 1.4 → PASS ✅
```

### APS Factors Analysis

| Factor | Impact | Implementation | Effect on APS |
|--------|--------|----------------|---------------|
| **Multiple Solutions** | ↓ Lower | 2-3 paths per level (11-50) | Reduces APS, increases fairness |
| **Path Planning** | ↑ Higher | Must see ahead, strategic | Increases APS, adds depth |
| **Limited Moves** | ↑ Higher | Tight move count (hard levels) | Increases APS, creates tension |
| **Obstacles** | ↑ Higher | Locked tiles, one-way paths | Increases APS moderately |
| **Hidden Info** | ↑ Higher | NONE (transparent design) | Minimizes unfair APS increase |
| **Time Pressure** | ↑↑ Higher | NONE (no timer) | Keeps APS manageable |

**Net Result:** Balanced design → APS 1.45 (sweet spot)

## 🚧 OBSTACLES & APS IMPACT

### 4 Obstacle Types

| Obstacle | Type | APS Impact | First Used | Complexity |
|----------|------|------------|------------|------------|
| **Limited Colors** | Constraint | ↓ Low (easier matches) | Level 1 | Tutorial |
| **Dead Ends** | Strategic | ↑ Low-Medium | Level 2 | Teaches planning |
| **Locked Tiles** | Constraint | ↑ Medium | Level 2 | Requires color matching |
| **One-Way Paths** | Constraint | ↑↑ High | Level 3 | Sequence planning |

### Obstacle Progression

```
Level 1-5:   No obstacles (pure onboarding)
Level 6-15:  Limited colors + Dead ends (teach planning)
Level 16-30: Add Locked tiles (strategic choices)
Level 31-50: All obstacles combined (mastery)
```

**APS Impact by Obstacle:**
- 0 obstacles: APS 1.1-1.3
- 1-2 obstacles: APS 1.4-1.6
- 3-4 obstacles: APS 1.7-1.9 (monetization zone)
